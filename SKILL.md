# Stop Slop ZH — 中文去 AI 味

写中文的时候别像在做完形填空——每个词应该是你选的，不是统计学选的。

> 完整词表：`references/phrases-zh.md`
> 严重度说明：`references/severity.md`
> 结构反模式：`references/structures.md`

**写的时候** — 落笔前过一遍脑子，别让这些词混进去。
**改的时候** — 拿到一段中文，逐条扫描替换。用户说"去掉 AI 味""太机器了""说人话"时触发。

## 底线

1. 开口就说事，别铺垫
2. 要强调就给证据，别堆形容词
3. 说完就走，别鞠躬

---

## 先看场景，再决定改多狠

| 场景 | 档位 | 说明 |
|------|------|------|
| 即时消息、对话回复 | minimal | 只砍 Tier 1 套话和谄媚。保留口语感。 |
| 技术摘要、变更日志、代码注释 | standard | 砍套话 + 渲染词。允许系统主语，不强制人称。 |
| 文档、README、教程 | standard | 同上。技术精度优先。 |
| 博客、公众号、社交媒体 | aggressive | 全部规则生效，两遍制工作流。 |
| 代码、配置、日志 | off | 不适用。 |

拿不准就按 standard 处理。

## Tier 1 高频词速查（默认替换）

按处理方式分组，只列高频的。完整词表见 `references/phrases-zh.md`。

### 直接删除类

删掉不丢东西，留着全是噪音。

**开场/过渡套话：**
值得注意的是 / 需要指出的是 / 不可否认的是 / 不难发现 / 不容忽视 / 众所周知 / 让我们一起来看看 / 接下来我将为你 / 在当今社会 / 随着……的不断发展 / 不得不说 / 诚然 / 深入探讨 / 具体来说 / 更重要的是 / 综上所述 / 总而言之 / 总的来说 / 总体来看 / 由此可见 / 换句话说 / 简而言之 / 归根结底 / 不言而喻 / 可以说 / 在此过程中 / 在这个过程中 / 简单的说 / 一句话总结 / 结论先说清楚 / 好，/ 行，（作为回复开场）

**过度强调：**
毋庸置疑 / 毫不夸张地说 / 值得深思 / 令人深思 / 引发思考 / 发人深省 / 不禁让人（想到/感叹）

**谄媚/元评论：**
好问题！/ 你说得很对 / 这是一个很好的观点 / 让我来为你解释 / 希望这对你有帮助 / 如果你有其他问题

**过度表达执行意愿（删掉，直接做事）：**
我已确认 / 我立马开始 / 要不要我…… / 如果你愿意…… / 如果你要…… / 只要你回复我…… / 你一回复我就…… / 你就确认一点

**虚假否定铺垫：**
不是……而是……（多数可删掉前半句）/ 我不猜 / 不靠猜 / 不瞎猜（删掉，说清楚你掌握了什么）/ 不躲 / 不藏 / 不绕 / 不逃（删掉，直接说）/ 说穿 / 说人话就是（删掉，直接说）

**正能量收尾模板：**
与其……不如积极拥抱…… / 让我们拭目以待 / 未来可期

### 替换为具体表达类

这些词什么都没说，换成真正在说的那句话。

**渲染性强调 → 说清楚具体是什么：**

| 禁用 | 处理 |
|------|------|
| 深刻的（影响/意义/变革）| 说清楚具体影响了什么 |
| 深远的（影响/意义）| 同上 |
| 不可磨灭的（贡献/印记）| 说清楚具体做了什么 |
| 至关重要 / 举足轻重 | "重要"或直接说为什么重要 |
| 令人瞩目 / 令人惊叹 | 给具体数据 |
| 意义非凡 / 具有重要意义 | 说清楚什么意义 |
| 前所未有 / 史无前例 | 给对比数据 |
| 颠覆性（的变革/创新）/ 范式转移 | 说清楚改变了什么 |
| 发挥着关键作用 | 说清楚怎么起作用 |
| 核心在于 / 关键在于 / 本质上 / 某种程度上 | 直接说或说清楚 |

**商业/互联网黑话 → 用普通词替换：**

| 禁用 | 替换 |
|------|------|
| 赋能 / 助力 | 帮、让……能 |
| 打造 | 做、建 |
| 抓手 | 方法、工具 |
| 闭环 | 完整流程 |
| 颗粒度 | 细节程度 |
| 对齐 / 拉齐 / 拉通 | 统一、一致 |
| 沉淀 | 积累、记录 |
| 痛点 | 问题 |
| 场景化 | 按场景 |
| 降本增效 | 省钱提速 |
| 底层逻辑 | 原因、原理 |
| 顶层设计 | 整体规划 |
| 体感 | 感觉、体验 |
| 心智 | 认知、印象 |
| 链路 | 流程、路径 |
| 触达 | 到达、联系到 |
| 透传 | 传递 |

**泛化的技术术语 → 用日常词替换：**

AI 把调试/运维术语用到日常对话里，像在写 postmortem。在技术报告（incident report、变更日志）中这些是标准术语，不替换。

| 禁用（日常对话中） | 替换 |
|------|------|
| 根因 | 原因、根本原因 |
| 落盘 / 已经落下去 | 保存、写入 / 已经做了 |
| 收口 / 收束 | 收尾、结束 |
| 收敛 / 收窄 | 缩小范围 |
| 兜底 | 保底处理 |
| 压实 | 落实、确认好 |
| 锁住 | 固定好、确认 |
| 更硬 / 硬写 / 偏硬 | 更严格 / 直接写死 / 太死板 |
| 口径 | 说法、措辞 |
| 这轮 | 这次 |
| 能吃 | 能接受、能处理 |
| 稳稳兜住 / 稳稳接住 | 处理好、搞定 |
| 砍一刀 | 删掉、去掉 |
| 接住 | 处理、应对 |
| 打掉问题 | 修好、解决 |
| 避免漂移 | 别跑偏 |
| 有点满 | 内容太多了 |
| 更稳 / 不稳 | 更可靠 / 不可靠 |
| 拆开看 | 分开说、逐个看 |
| 对上了 / 坐实了 | 吻合、一致 / 确认了 |
| 抓到的现象 | 发现的问题 |
| 做一个更硬的排除法 | 用排除法再查一遍 |
| 把差异收窄了 | 缩小了差异 |

**夸张动词 → 用平实动词替换：**

| 禁用 | 替换 |
|------|------|
| 补一刀 | 再加一点、补充 |
| 更狠 / 狠一点 | 更彻底、更严格 |
| 狠狠干 | 删掉 |
| 打坏 | 破坏、损坏 |
| 拍脑门 | 随便决定（说清楚是谁） |
| 抠出来 / 揪出来 | 找出来、定位 |

**自媒体流水线用语 → 删掉或换成普通词：**

| 禁用 | 处理 |
|------|------|
| 保姆级（教程/攻略）| 删掉或换成"详细" |
| 硬核（干货/分析）| 删掉 |
| 干货 | 删掉 |
| 拆解 | 分析、讲解 |
| 梳理 | 整理 |
| 盘点 | 列举、介绍 |
| 避坑 / 踩坑 | 注意、别犯的错 |
| 一文读懂 / 万字长文 / 建议收藏 / 强烈推荐 / 划重点 | 删掉 |
| 绝绝子 / 谁懂啊 / 真的会谢 / 姐妹们 / 狠狠（XX了）| 删掉 |

**无源引用 → 补具体来源或删掉：**
研究表明…… / 数据显示…… / 有专家指出…… / 业内人士认为…… / 据报道……

## Tier 2：同段出现 2+ 个时标记

单独用正常，聚集出现是 AI 味信号。保留最贴切的一个，其余替换或改写句式。

**连接词：** 然而 / 此外 / 与此同时 / 不仅……而且…… / 一方面……另一方面…… / 尽管如此 / 事实上 / 实际上 / 在此基础上 / 进一步地 / 恰恰 / 正是 / 无疑 / 由此可以看出 / 不外乎

**修饰词：** 显著 / 有效 / 全面 / 积极 / 持续 / 进一步 / 充分 / 切实 / 可谓 / 堪称 / 追根溯源

**单音节命令词（编程场景密集堆砌时标记）：** 补 / 接 / 核 / 进 / 顺 / 落 / 坏 / 跑

## Tier 3：全文密度高时标记

常见词，只在全文中饱和使用时才有 AI 味。用同义词替换部分出现。

重要 / 关键 / 核心 / 基础 / 创新 / 优化 / 提升 / 推动 / 加强 / 确保 / 实现 / 促进

## 结构层：词表抓不到的问题

光查词不够，以下是句子和段落层面的 AI 痕迹，详见 `references/structures.md`：

1. **虚假主语** — 给无生命事物安人类动作（"该框架赋能了……"），改为具体主语
2. 被动语态堆砌 — 连续被动句隐藏执行者，改为主动句
3. 二元对比假戏剧 — "不是 X，而是 Y"，直接说 Y
4. **三件套列举** — AI 偏爱三个一组，两个或一个往往更自然
5. "首先…其次…最后…"机械排列 — 打散，用自然过渡
6. 总结式收尾 — 前面说清楚了就不用"综上"再说一遍
7. 正能量收尾 — 不管前面说了什么最后必须上价值，删掉
8. 对称填充 — 为了"平衡"硬凑对仗，没有信息增量
9. 加粗滥用 — 机械地给每个要点加粗
10. **分条列点强迫症** — 简单回复也 1. 2. 3.，用自然段落
11. 句长均匀 — 连续 5 句话最长和最短差距不到 2 倍，节奏太平
12. 翻译腔 — 长定语链、被动语态堆砌、"基于……""通过……来……""对于……而言""在……方面""从……的角度来看"

## 审阅流程

### minimal / standard

一遍过，按这个顺序查：

1. **开场** — 有没有套话？直接删
2. **黑话和泛化术语** — 替换为日常用词
3. **夸张动词和自媒体用语** — 技术回复重点检查
4. **执行意愿过度表达** — 删掉所有"要不要我""如果你愿意"
5. **收尾** — 删掉谄媚句和正能量模板
6. **翻译腔** — 长定语链、被动语态堆砌
7. **通读** — 每句话是人类会说的吗？

### aggressive（过两遍）

第一遍：按全部规则扫一遍，包括结构层。改表达，不改意思。

第二遍：读改完的文本，问自己"还有哪里一眼看出是 AI 写的？"，找到就修。

## 误杀防护

命中词表但不该改的情况：

1. **引用原文** — 他人原话或文档原文保留原样
2. **术语定义** — 当这个词本身是被讨论的对象（如"什么是赋能"）
3. **代码/配置** — 技术名词、变量名、API 名称不改
4. **特定行业标准术语** — 金融的"杠杆"、技术报告的"根因""收口"
5. **系统主语的技术描述** — "网关返回 504""缓存过期"是合理的
6. **真人网络用语** — 博主在具体经历后自然使用"踩坑""谁懂啊"，有具体细节支撑时不是 AI 腔
